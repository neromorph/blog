## Learn the boring tools

`grep`, `find`, `xargs`, `awk`, and shell history solve more daily problems than most new tools.

## Save repeatable commands

If you run a command twice, consider saving it in a script, Makefile, or project README.

## Prefer readable aliases

Aliases should remove typing, not hide meaning.
